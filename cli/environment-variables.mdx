## Overview

Codex Multi-Auth supports environment variable overrides for most configuration settings. Environment variables take precedence over settings.json values.

---

## Directory and Path Configuration

<ParamField path="CODEX_HOME" type="string">
  Override the Codex home directory.
  
  **Default:** `~/.codex`
  
  **Example:** `export CODEX_HOME="/custom/path"`
</ParamField>

<ParamField path="CODEX_MULTI_AUTH_DIR" type="string">
  Override the multi-auth storage directory.
  
  **Default:** `~/.codex/multi-auth`
  
  **Example:** `export CODEX_MULTI_AUTH_DIR="/custom/multi-auth"`
</ParamField>

<ParamField path="CODEX_MULTI_AUTH_CONFIG_PATH" type="string">
  Override the config file location (legacy).
  
  **Note:** This is for legacy config.json files. Modern installations use settings.json.
</ParamField>

<ParamField path="CODEX_CLI_ACCOUNTS_PATH" type="string">
  Override the Codex CLI accounts file path.
  
  **Default:** `~/.codex/accounts.json`
</ParamField>

<ParamField path="CODEX_CLI_AUTH_PATH" type="string">
  Override the Codex CLI auth file path.
  
  **Default:** `~/.codex/auth.json`
</ParamField>

## Core Features

<ParamField path="CODEX_MODE" type="boolean" default="1">
  Enable CODEX_MODE for Codex CLI parity.
  
  **Values:** `1` = enabled, `0` = disabled
  
  **Example:** `export CODEX_MODE=1`
</ParamField>

<ParamField path="CODEX_TUI_V2" type="boolean" default="1">
  Enable the v2 terminal user interface.
  
  **Values:** `1` = enabled, `0` = disabled
</ParamField>

<ParamField path="CODEX_TUI_COLOR_PROFILE" type="string" default="truecolor">
  TUI color profile mode.
  
  **Options:** `truecolor`, `ansi16`, `ansi256`
  
  **Example:** `export CODEX_TUI_COLOR_PROFILE=ansi256`
</ParamField>

<ParamField path="CODEX_TUI_GLYPHS" type="string" default="ascii">
  TUI glyph rendering mode.
  
  **Options:** `ascii`, `unicode`, `auto`
  
  **Example:** `export CODEX_TUI_GLYPHS=unicode`
</ParamField>

## Boolean Value Format

For boolean environment variables:
- `1` = enabled/true
- `0` = disabled/false

Any other value is ignored and the default from settings.json or built-in default is used.

## Examples

### Basic Configuration

```bash
# Enable debug logging
export DEBUG_CODEX_PLUGIN=1

# Use custom directory
export CODEX_MULTI_AUTH_DIR="/custom/path"

# Increase timeout
export CODEX_AUTH_FETCH_TIMEOUT_MS=120000
```

### Development Setup

```bash
# Enable all debugging
export DEBUG_CODEX_PLUGIN=1
export CODEX_PLUGIN_LOG_BODIES=1
export CODEX_PLUGIN_LOG_LEVEL=debug

# Disable production features
export CODEX_AUTH_PROACTIVE_GUARDIAN=0
export CODEX_AUTH_LIVE_ACCOUNT_SYNC=0
```

### Performance Tuning

```bash
# Enable fast mode
export CODEX_AUTH_FAST_SESSION=1
export CODEX_AUTH_FAST_SESSION_STRATEGY=always

# Parallel probing
export CODEX_AUTH_PARALLEL_PROBING=1
export CODEX_AUTH_PARALLEL_PROBING_MAX_CONCURRENCY=4
```
